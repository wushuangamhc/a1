# 02 小黑屋与战斗闭环

## 目标

围绕小黑屋测试场，把 `移动 -> 发射 -> 命中 -> 死亡 -> 复活` 闭环做稳定。

## 涉及目录

- `game/game_mode.ts`
- `game/systems/combat_system.ts`
- `game/systems/pickup_system.ts`
- `game/systems/map_system.ts`
- `content/panorama/src/hud.tsx`

## 验收标准

- 三个英雄原型都能触发各自射击逻辑
- 护盾祝福能正确抵挡一次伤害
- 死亡后能快速复活到指定出生点
- HUD 能反映当前英雄、祝福、符文和击杀数据

## 给 Kimi 的执行提示词

你现在负责把这个 Dota 2 UGC 项目的“小黑屋战斗闭环”做稳定。请遵守下面规则：

1. 不要扩 scope 去做正式美术或复杂演出。
2. 所有新增逻辑优先复用 shared 的类型和事件。
3. 如果某个 Dota API 是否存在不确定，优先保守实现并在结果里标记风险点。
4. 不要破坏现有 FFA 主循环。

你的工作重点：
- 检查 combat_system 的命中判定、回旋、护盾消耗
- 检查 respawn 与 scoreboard 同步
- 让 HUD 的 debug 行为更适合小黑屋测试
- 最后输出：改动文件、验证方法、未验证风险
